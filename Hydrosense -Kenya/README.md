# HydroSense-Kenya

A scientific-computing system for smart irrigation, water-balance simulation, and climate-aware
decision support, built for ICS 2207 (Scientific Computing).

## Problem

Given daily weather records and three farm zones (tomato, kale, maize) with noisy/incomplete sensor
data, estimate soil-water balance, detect water stress, simulate future soil moisture under rainfall
uncertainty, and recommend an irrigation schedule that minimizes water use without exposing crops to
moisture stress.

pip install -r requirements.txt

## Running the Notebooks

Open `notebooks/` in Jupyter or Google Colab and run levels in order (1 -> 6). Each notebook imports
shared functions from `src/` via:

```python
import sys, os
sys.path.insert(0, os.path.abspath("../src"))
```

## Running the Tests

The test suite uses `pytest`:

```bash
pytest tests/ -v
```

If `pytest` is not available in your environment, every test file can also be run as a plain script
(`python tests/test_root_finding.py`) since the test functions take no arguments and assert internally.

## Core Model

Discrete water balance:

```
S_(t+1) = S_t + R_t + I_t - ET_t - D_t
```

Simplified evapotranspiration:

```
ET = max(0, 0.12*T + 0.35*W + 2.4*Solar - 0.025*H)
```

## Datasets

| File | Purpose |
|---|---|
| `weather_daily.csv` | Daily rainfall, temperature, humidity, wind speed, solar index (March 2026, with intentional missing values and an outlier/sensor-fault temperature reading). |
| `soil_sensor_data.csv` | Daily noon soil moisture, tank level, pump flow, and pump power for three zones, including a missing reading, an implausible tank-level spike, and a zero-flow/"CHECK" pump fault. |
| `crop_zone_parameters.csv` | Per-zone crop type, area, minimum/target moisture, field capacity, and drainage coefficient. |
